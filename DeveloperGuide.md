# AI Step Tracker — Developer Guide

## Overview

This document explains the architecture, components, workflows, deployment process, and implementation details of the AI Step Tracker application.

The goal of this guide is to help future developers:

- Understand the application architecture
- Modify UI and business logic safely
- Extend the application with new features
- Debug deployment and API issues
- Maintain Google Sheets integration
- Improve analytics and AI insights

---

# 1. Application Overview

The application is a cloud-synced AI-powered step tracker.

Users manually enter step counts from:

- Samsung Health
- Zepp App

The application:

- Combines both sources
- Saves data to Google Sheets
- Loads historical data
- Displays charts and analytics
- Tracks streaks and achievements
- Shows AI-generated insights
- Works on desktop and Android

# 7.5 Important CORS Fix

```javascript
headers:{
  "Content-Type":"text/plain;charset=utf-8"
}
```

Why required:

Google Apps Script has issues with:

```text
application/json
```

Using:

```text
text/plain
```

avoids browser preflight/CORS failures.

# 8. Google Apps Script Explained

Apps Script acts as backend API.

Purpose:

- Read Google Sheet
- Write/update rows
- Return JSON data

---

# 8.1 normalizeDate()

Critical fix.

Purpose:

Google Sheets stores dates as Date objects.

Frontend sends:

```text
2026-05-17
```

Normalization ensures equality works.

Without this:

❌ duplicate rows created

---

# 8.2 doPost(e)

Purpose:

Handle saves from frontend.

Flow:

```text
Frontend POST
  ↓
doPost(e)
  ↓
Find matching date
  ↓
Update row OR append row
```

---

# 8.3 Row Update Logic

```javascript
if(rowDate === incomingDate)
```

If match exists:

```javascript
setValues()
```

Otherwise:

```javascript
appendRow()
```

# 8.4 doGet()

Return all sheet rows as JSON.

# 10. Cache Busting

GitHub Pages caches JS/CSS aggressively.

Solution:

```html
<script src="app.js?v=2"></script>
```

Increment version after updates.

# 11. Common Issues

## 11.1 CORS Errors

Cause:

- opening HTML directly
- wrong Apps Script permissions

Fix:

- use GitHub Pages
- deploy Apps Script as Anyone

---

## 11.2 Duplicate Rows

Cause:

Date mismatch between frontend and Sheets.

Fix:

```javascript
normalizeDate()
```

---

## 11.3 Old Version Still Showing

Cause:

Browser cache.

Fix:

```text
Ctrl + Shift + R
```

or increment JS/CSS versions.
